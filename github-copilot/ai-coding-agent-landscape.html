Honest fit

When each one actually wins

Every tool below has a real niche. The question is which niche looks like your team.

GitHub Copilot cloud agent

First-party · Copilot Business/Enterprise
Best fit: org already on Copilot, many repos, governance in GitHub, platform team wants agent adoption off their queue.
Worst fit: you need a specific non-Anthropic non-OpenAI model, or you've explicitly rejected Copilot for procurement reasons.
  • Zero per-repo files required
  • First-party bot identity in git log
  • Audit trail in GitHub's existing surfaces
  • Uses Actions minutes + Copilot premium requests

Claude Code Action

Anthropic · GitHub App + workflow
Best fit: team already trusts Claude for code, wants governance in GitHub Actions, OK shipping a workflow file per repo.
Worst fit: 50+ repos where per-repo workflow fan-out is painful, or org policy blocks third-party GitHub Apps.
  • Workflow YAML + ANTHROPIC_API_KEY per repo
  • Responds to @claude in issue/PR comments
  • Runs on your own GitHub runners
  • Supports Bedrock, Vertex AI, Microsoft Foundry

OpenAI Codex in GitHub

External SaaS · ChatGPT account
Best fit: teams already in ChatGPT for dev work, want @codex review on PRs without writing Actions YAML, OK with governance in ChatGPT.
Worst fit: regulated orgs that need every agent action auditable inside GitHub's own logs, not a vendor dashboard.
  • No workflow file needed
  • Per-repo toggle in ChatGPT Codex settings
  • Optional AGENTS.md for review guidelines
  • Auto-review mode for every PR opt-in

Cursor Cloud Agents

External SaaS · Cursor team plan
Best fit: Cursor IDE shop wanting the same agent UX in cloud; teams using Slack/Linear as primary issue surfaces, not GitHub.
Worst fit: teams not on Cursor IDE; orgs that restrict third-party R/W GitHub integrations.
  • Triggered from Cursor web, Desktop, Slack, Linear, GitHub, API
  • Models always run in Max Mode
  • MCP + hooks support (.cursor/hooks.json)
  • Billed at API pricing per model

What this piece is not

  • Not a capability ranking. None of these cards claim one agent writes better code than another — that depends on your stack and benchmark.
  • Not a pricing comparison. All four change pricing on months-long cycles; any dollar figure here would age fast.
  • Not vendor-neutral on origin. The author is a Microsoft employee. The framing tries to be honest about where Copilot doesn't win.
